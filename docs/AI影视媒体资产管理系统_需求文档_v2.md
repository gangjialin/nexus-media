## 一、项目背景与目标

### 1.1 当前痛点

公司影视制作流程中，以下瓶颈日益突出：

| 痛点 | 现状 | 影响 |
|------|------|------|
| **剧本版本混乱** | 微信/飞书来回传几十版，不清楚哪个是定稿 | 制作组拿错版本返工 |
| **审阅意见散落** | 群聊、私聊、会议各说各的，无跟踪 | 修改意见遗漏，责任不清 |
| **素材检索困难** | NAS 上数 TB 素材仅靠文件夹名 + 人脑记忆 | "去年那个夕阳镜头在哪"——无人知晓 |
| **进度不透明** | 各组进度全靠口头汇报 | 发现问题时已延误工期 |
| **重复制作** | 不知其他组已做过类似内容 | 人力与时间浪费 |

### 1.2 项目目标

构建一套 **AI 驱动的影视媒体资产管理系统**，实现：

- **资产集中化**：剧本、视频、音频、图片统一管理，给 NAS 加上"检索大脑"
- **流程在线化**：剧本分发 → 制作 → 审阅 → 定稿，全链路在线协作
- **审阅可追踪**：每条修改意见有来源、有状态、有确认
- **进度可视化**：从导演到制片，全局看板一目了然
- **智能化辅助**：AI 赋能分镜、概念图、标签、搜索、分析

## 二、行业参考分析

### 2.1 阿里云智能媒体服务（ICE / IMS）MediaAI

阿里云将原"智能媒体生产（ICE）"升级为 **智能媒体服务（IMS）**，覆盖媒体采集、媒资管理、内容生产、制作和分发全链路。

#### AI 内容理解能力

| 能力类别 | 具体功能 | 可借鉴点 |
|---------|---------|---------|
| **文字识别** | OCR、字幕提取、ASR 语音转文字 | 剧本/素材中文字自动提取 |
| **视觉识别** | 人脸识别、人物角色识别、物体识别、地标识别 | 演员、道具、场景自动标注 |
| **场景分析** | 场景识别、动作识别、表情识别 | 视频内容自动分类 |
| **内容审核** | 色情/暴恐/涉政/违规识别 | 素材合规自动过滤 |
| **智能标签** | 视频分类、节目识别、台标识别 | 上传后自动打标签 |

#### AIGC 与智能生产（2025-2026）

| 能力 | 说明 | 借鉴价值 |
|------|------|---------|
| **一键成片** | AI 根据素材自动剪辑成片 | 远期可做预告片自动生成 |
| **视频翻译** | 多语言字幕/配音自动生成 | 海外发行需求 |
| **多模态检索** | 通义千问大模型自然语言搜索 | AI 语义搜索核心参考 |
| **智能封面** | 自动选取最佳视频帧作为封面 | 直接可用 |
| **智能横转竖** | 适配短视频平台比例 | 多平台分发需求 |
| **绿幕抠图** | 人像/背景分离 | 后期制作辅助 |

**关键洞察**：阿里云已从传统 AI 标签走向 **LLM 驱动的多模态理解**，2026 冬奥会采用通义千问实现 8PB 媒体资产的 AI 自动打标与自然语言检索。

### 2.2 AWS 媒体资产管理（MAM）

AWS 将 MAM 定义为"组织图像、音频和视频等媒体文件的全流程管理体系"。

#### 核心架构理念

```
采集 → 摄入 → 处理 → 管理 → 分发 → 归档
```

| 阶段 | AWS 方案 | 可借鉴点 |
|------|---------|---------|
| **摄入** | 自动触发转码、元数据提取 | 上传即处理的事件驱动架构 |
| **处理** | Lambda 无服务器工作流编排 | 转码/分析等任务队列设计 |
| **管理** | Amazon S3 分层存储 + 自定义元数据 | 文件与元数据分离存储 |
| **分发** | CDN + 代理文件（Proxy） | 高码率主文件 → 低码率代理预览 |
| **归档** | S3 Glacier 冷存储 | 历史项目自动归档策略 |

#### AI 集成方案

| AWS 服务 | 能力 | 对应到本系统 |
|---------|------|-------------|
| **Amazon Rekognition** | 对象/场景/人脸/文字识别 | AI 标签 + 智能搜索 |
| **Amazon Transcribe** | 语音转文字 | 台词提取、字幕生成 |
| **Amazon Translate** | 多语言翻译 | 剧本/字幕翻译 |

#### 2025 年趋势

- **AI 赋能的资产发现**：ML 自动帧级精确打标
- **代理文件（Proxy）工作流**：高码率主文件存 NAS，低码率代理用于在线预览
- **无服务器化**：事件驱动、按需扩展

### 2.3 三方对比与启示

| 维度 | 用户原始需求 V1 | 阿里云 ICE MediaAI | AWS MAM |
|------|----------------|-------------------|---------|
| **核心定位** | 影视制作团队协作 | 全链路媒体处理平台 | 云原生资产管理框架 |
| **AI 侧重点** | 分镜/概念图/标签/搜索 | 内容理解/AIGC/审核 | Rekognition + Transcribe |
| **架构理念** | NAS + 检索层 | 全托管云服务 | 无服务器事件驱动 |
| **存储策略** | 文件存 NAS，元数据存 DB | 云存储（OSS） | S3 分层存储 + 归档 |
| **审阅协作** | 核心特色功能 | 较弱 | 较弱 |
| **对外分发** | 外部分享链接 | 多平台分发 | CDN + 转码管道 |

**启示**：本系统应取三者之长——**以用户 V1 的影视制作协作流程为核心**，吸收阿里云的多模态 AI 理解能力和 AWS 的架构理念（代理文件、事件驱动、存储分层）。

## 三、用户角色与权限体系

### 3.1 角色定义

| 角色 | 权限层级 | 核心职责 | 可访问范围 |
|------|---------|---------|-----------|
| **管理员** | 系统级 | 系统配置、角色管理、存储管理、日志审计 | 全部 |
| **导演 / 总导演** | 项目级 | 剧本审批、分发任务、终审成品、全局进度 | 所有项目 |

| **外部合作方** | 受限级 | 查看被分享内容、限时下载 / 仅在线预览 | 被分享的资源 |

## 五、AI 智能引擎详述

### 5.1 整体架构

```
用户上传素材 / 导入剧本
        │
        ▼
┌─────────────────────────────────┐
│          AI 任务调度器           │
│   (RabbitMQ + Celery Worker)    │
└──────┬────────────┬─────────────┘
       │            │
       ▼            ▼
┌────────────┐ ┌──────────────┐
│  实时任务   │ │  异步批量任务  │
│ (分镜/概念图)│ │ (标签/分析/索引)│
└────────────┘ └──────────────┘
```

### 5.2 AI 功能清单

| 功能 | 阶段 | 技术方案 | 说明 |
|------|------|---------|------|
| **AI 分镜生成** | 一期 | LLM（GPT-4 或通义千问）+ 提示词工程 | 输入剧本段落 → 输出结构化分镜表（镜头号、景别、运镜、画面描述、预估时长） |
| **AI 概念图** | 一期 | Stable Diffusion / Midjourney API | 输入文字描述 → 生成场景参考图 |
| **智能封面** | 一期 | 关键帧评分算法 | 自动选取最佳帧作为视频封面 |
| **自动标签** | 二期 | 多模态模型（CLIP + 分类器） | 上传后自动分析内容并生成标签 |
| **ASR 语音转文字** | 二期 | Whisper 或阿里云 ASR | 视频对白自动转录，生成可搜索文本 |
| **人脸/物体识别** | 二期 | YOLO + FaceNet | 识别演员、道具、场景元素 |
| **OCR 文字提取** | 二期 | PaddleOCR / Tesseract | 提取视频内字幕、标题文字 |
| **语义搜索** | 二期 | 向量嵌入（Embedding）+ Milvus/FAISS | 用自然语言搜索素材画面内容 |
| **内容审核** | 二期 | 多级审核模型 | 色情/暴力/政治敏感内容自动标记 |
| **智能横转竖** | 二期 | FFmpeg + AI 裁切 | 适配抖音/快手等竖屏比例 |
| **剧本分析** | 二期 | LLM + NLP 分析 | 角色出场统计、对白密度、情绪曲线 |
| **场景识别** | 二期 | 视频场景分割算法 | 自动将长视频拆分为不同的场景片段 |
| **以图搜视频** | 三期 | CLIP 视觉嵌入 | 上传参考图片→找到视频中相似画面 |
| **自动字幕生成** | 三期 | ASR + 翻译管道 | 自动生成 SRT 字幕，支持多语言 |
| **音乐节奏检测** | 三期 | 音频分析算法 | 检测 BGM 节奏点，辅助剪辑匹配 |

### 5.3 AI 数据流

```
素材上传
   │
   ▼
[AI 分析管道]
   ├─ 视频 → 场景分割 → 关键帧提取 → 人脸/物体/OCR/ASR
   ├─ 音频 → 语音转文字 → 说话人分离 → 关键词提取
   ├─ 图片 → EXIF 读取 → 内容识别 → 风格分类
   └─ 剧本 → 角色提取 → 场次分析 → 情绪曲线
             │
             ▼
      [向量嵌入 + 标签]          [结构化元数据]
           │                          │
           ▼                          ▼
      [向量数据库]              [PostgreSQL]
      (Milvus/FAISS)           (关系型元数据)
           │
           ▼
      [语义搜索接口]
```

### 5.4 AI 模型选型建议

| 场景 | 推荐方案 | 备选方案 |
|------|---------|---------|
| 语音转文字 | Whisper（本地部署） | 阿里云 ASR / 讯飞 |
| 图像识别 | YOLOv11 + CLIP | 阿里云 Rekognition |
| 语义搜索 | text2vec + Milvus | OpenAI Embedding + Pinecone |
| 文本生成（分镜/分析） | 通义千问（阿里云） | GPT-4o / Claude |
| 概念图生成 | Stable Diffusion（本地/API） | Midjourney API |
| 人脸识别 | FaceNet + ArcFace | 阿里云人脸搜索 |
| OCR | PaddleOCR | Tesseract 5 |

> **建议**：一期以 LLM API 调用为主，快速验证效果；二期根据调用量评估是否引入本地模型。

### 6.2 部署方案对比

| 维度 | 方案 A：本地部署 | 方案 B：云部署（推荐初期） | 混合方案（推荐） |
|------|----------------|--------------------------|-----------------|
| 应用服务器 | 公司本地服务器 | 阿里云 ECS / 腾讯云 CVM | 云服务器 |
| 媒体存储 | NAS（内网访问） | OSS / S3 | NAS（主文件）+ 云（代理文件） |
| 数据库 | 本地 PostgreSQL | 云数据库 RDS | 云数据库 RDS |
| AI 算力 | 本地 GPU 服务器 | 阿里云 PAI / 云端 GPU | 云端 GPU 按需 |
| 网络 | 内网（高速） | 公网（需 CDN） | 内网 NAS + 公网 API |
| 维护 | 需 IT 运维 | 免运维 | 低维护 |
| 数据安全 | 数据完全本地 | 数据在云端 | 素材在本地，元数据在云 |

**推荐初期方案**：混合部署
- 核心服务部署在云服务器（弹性、免运维）
- 素材仍存本地 NAS（不迁移已有数 TB 数据）
- 代理文件（低码率预览）存云对象存储
- AI 计算使用云端 GPU（按需付费，不用时关停）
- 外网访问通过云服务器中转 → 代理文件直接从云存储分发

### 6.3 代理文件（Proxy）工作流

```
NAS 原始素材（高码率 / ProRes / RAW）
        │
        ▼
[转码服务] ───→ 代理文件（H.264 1080p / 低码率）
        │                      │
        ▼                      ▼
   本地剪辑              云对象存储
   (达芬奇/Premiere)      (在线预览用)
```

- 上传时自动触发转码
- 在线预览播放代理文件，不占用 NAS 带宽
- 最终输出时从 NAS 取原始高质量文件

### 8.2 NAS 集成方案

```
NAS 现状
  ├── /projects/ProjectA/              ← 已有数据不动
  │       ├── footage/                 ← 拍摄素材

Nexus Media 系统
```

## 十一、附录：术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| 媒体资产 | Media Asset | 影视制作中产生的各种数字文件，包括视频、音频、图片、3D 模型等 |
| 元数据 | Metadata | 描述数据的数据，如视频的时长、分辨率、编码、标签等 |
| 代理文件 | Proxy File | 原始高码率素材的低分辨率/低码率副本，用于预览和离线剪辑 |
| 转码 | Transcode | 将媒体文件从一种编码格式转换为另一种 |
| 断点续传 | Resume Upload | 大文件上传中断后可从断点继续，无需重传 |
| 分镜 | Storyboard | 用画面和文字描述每个镜头的拍摄方案（景别、运镜、构图、时长） |
| 向量嵌入 | Vector Embedding | 将文字、图片等内容转换为数学向量，用于语义相似度搜索 |
| 向量数据库 | Vector Database | 专门存储和检索向量数据的数据库，如 Milvus、FAISS |
| RBAC | Role-Based Access Control | 基于角色的访问控制，按角色设定权限 |
| 甘特图 | Gantt Chart | 项目时间线管理图表，横轴时间纵轴任务 |
| 看板 | Kanban | 可视化任务管理，任务卡片在"待办/进行中/已完成"列间流动 |
| 水印 | Watermark | 画面上的半透明标识文字或图案，防未经授权传播 |
| ASR | Automatic Speech Recognition | 自动语音识别，将语音转为文字 |
| OCR | Optical Character Recognition | 光学字符识别，从图片/视频中提取文字 |
| AIGC | AI Generated Content | AI 生成内容，如 AI 分镜、AI 概念图 |
| 多模态 | Multimodal | 同时处理多种类型数据（文本、图像、音频、视频）的能力 |
| 冷热分层 | Tiered Storage | 按访问频率将数据放在不同性能和成本的存储介质上 |
